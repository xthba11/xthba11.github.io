欢迎来到我的技术博客。这里会记录我在 STM32 固件、FreeRTOS、LVGL、传感器驱动、Linux 系统编程和 C 语言网络编程中的实践笔记。

我希望这个站点不只是“代码片段仓库”，而是能把一个问题从背景、设计取舍、实现细节、调试方法一路讲清楚。写不确定的地方时，我会标出来；已经验证过的地方，我会尽量附上环境、命令、日志或代码路径。

## 当前主项目：RidePulse

我现在重点整理的是 **RidePulse 自行车码表**。它来自我之前做的一套 STM32F411 智能手表固件，原工程已经包含：

- FreeRTOS 多任务框架
- LVGL 显示和触摸交互
- 心率、气压、温湿度、MPU6050 等传感器服务
- 外部 Flash 与 LittleFS
- USART + Ymodem OTA
- 低功耗状态机
- 软件看门狗 + 硬件 IWDG

接下来我会把它改造成自行车码表，补齐轮速采集、速度/里程计算、骑行状态机、码表 UI 和骑行记录保存。这个项目会作为博客里最主要的项目实战线索。

## 内容主线

### MCU 与外设开发

这一部分关注从硬件到固件的完整链路：

- GPIO、UART、I2C、SPI、CAN、PWM、ADC、DMA 等常见外设
- STM32 工程结构、启动文件、链接脚本、时钟树、Bootloader
- 传感器、电机、蓝牙模块、显示屏等实际模块调试
- J-Link、ST-Link、SWD、串口日志和逻辑分析仪的使用方法
- 外部 Flash、LittleFS、OTA 和低功耗这类产品化细节

### RTOS 与实时系统

RTOS 不是简单地把逻辑拆成几个任务，而是要考虑实时性、资源竞争和故障恢复：

- FreeRTOS 任务划分、优先级设计、消息队列、事件组
- 中断与任务协作
- 栈溢出、死锁、优先级反转等常见问题
- 看门狗、异常日志、掉电保护等工程化机制

### Linux 驱动与系统编程

Linux 方向会从用户态到内核态逐步展开：

- 字符设备、platform driver、设备树、GPIO/I2C/SPI 子系统
- procfs、sysfs、ioctl、poll/select/epoll
- Makefile、交叉编译、GDB、perf、ftrace
- 驱动调试、内核日志、崩溃定位

### C 语言网络编程

网络专题偏向工程实践：

- TCP/UDP socket 编程
- epoll 高并发服务模型
- 粘包/半包、心跳、重连、超时管理
- Wireshark、tcpdump 抓包分析
- 协议栈中常见的数据结构和算法

## 推荐阅读路径

如果你刚进入嵌入式方向，可以按下面顺序阅读：

1. RidePulse 项目总览：先看一个真实工程怎么拆
2. FreeRTOS 任务划分：理解任务、队列、互斥锁和看门狗
3. 轮速采集与速度计算：把码表核心闭环跑起来
4. LVGL 码表 UI：把业务数据稳定显示出来
5. LittleFS 骑行记录：把一次骑行保存到外部 Flash
6. STM32 工程结构设计：整理 MCU 工程分层
7. Linux 字符设备驱动入门：切到嵌入式 Linux 方向
8. C 语言 epoll 服务端实践：补系统编程和网络调试能力

## 我会怎样写文章

每篇文章尽量包含这些部分：

- 测试环境：MCU、开发板、工具链、系统版本、依赖库版本
- 问题背景：为什么要做这个，来自哪个项目或实验
- 设计思路：关键模块怎么拆，接口怎么定，为什么这样取舍
- 代码实现：贴关键路径代码，而不是堆完整源码
- 验证方法：如何编译、如何运行、期望看到什么日志
- 常见坑：真实开发中容易踩的地方，以及我会怎么定位
- 复盘：这次之后形成什么检查清单或工程习惯

嵌入式开发很迷人的地方在于，它把软件、硬件、时序、协议和工程经验揉在一起。这个博客会尽量保留这些细节，而不是只留下整理过的“正确答案”。
